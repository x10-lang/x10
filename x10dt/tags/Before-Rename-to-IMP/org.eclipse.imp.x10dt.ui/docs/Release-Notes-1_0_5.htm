X10DT 1.0.5 Release Notes

There is now an updated version (1.0.5) of X10DT available on the update server.

See the section "Installing X10DT" below for details on how to get it.

 

New and Noteworthy:

·        There is an X10 Preferences page. After installing, but before using the tools, go to that page and indicate where your x10.common directory is and which X10 configuration file (e.g. x10.common/etc/standard.cfg) you're using.

·        There is now an X10 launch configuration for both Run and Debug. You can access these in the usual way, by "Run As..." -> "X10 Application" or "Debug As..." -> "X10 Application". There is also a keyboard short-cut (Alt-X X).

Each instance of a launch configuration will by default use the X10 runtime that is located in the x10.runtime directory that is a sibling of the x10.common directory specified in the X10 Preferences page. To modify it, see the "X10 Runtime" page in the launch configuration dialog.

For now, this is a single-VM run.

·        To enable the X10 builder on a project, right-click on the project in the Package Explorer and select "X10" -> "Enable Builder". You will probably have to then invoke "Project" -> "Clean..." to get the builder to run the first time.

·        To enable the JikesPG builder on an existing project, right-click on the project in the Package Explorer and select "Enable JikesPG Builder".

·        The builder works more of the time.

IMPORTANT: See the Release Notes (below) about setting the build-time classpath.

IMPORTANT: See the Release Notes (below) about inter-file dependencies.

 

·        The outline view no longer becomes blank when a syntax error is introduced in the X10 source.

·        "Auto-indentation" now works in a manner similar to that for ordinary Java source files.

If you hit TAB while at the beginning of a line, the line will be indented appropriately according to its nesting inside parent constructs. There is no "auto-formatting" as yet (i.e. like what you get in Java source files by invoking "Source" -> "Format"). To be more precise, you *can* invoke that function, but you'll get ordinary Java formatting, which will probably get quite confused by the X10 language constructs.

·        X10DT should now install and work on Linux.

I've compiled the jikespg generator executable on Gentoo under gcc 3.3.6 and glibc 2.3.5. If this doesn't work on your system, let me know and I'll work something out.

·        The JikesPG IDE now shows more elements in the Outline View, and the builder is better behaved wrt error messages and failing to find the executable. The JikesPG builder should now also run under Linux.

 

Release Notes:

·        Any X10 code that you actually want to compile and run from within Eclipse *must* be hosted inside an Eclipse "Java Project". This can be an existing Java project.

IMPORTANT: You must add the appropriate x10.runtime jar to the "Java build path" for the Java project. If you don't, you'll get errors like "Class not found: x10.lang.Object" when compiling.

To do this:

o       Go to "Project" -> "Properties" (or select "Properties" from the context menu of the project in the Pkg Explorer or Navigator) and select "Java Build Path" page.

o       On the right-hand side of the dialog, select "Libraries".

 

Now, if you want to just use the x10.runtime that's included in X10DT (up to date as of 12/15/2005), do the following:

  1. Click "Add Variable"
  2. Select "ECLIPSE_HOME"
  3. Click "Extend..."
  4. Expand the "plugins" folder and select "x10.runtime_1.0.5.jar" (near the bottom).
  5. Click "OK" in the Variable Extension dialog
  6. Click "OK" in the "New Variable Classpath Entry" dialog
  7. Click "OK" in the project properties dialog

 

If instead you want to use another x10.runtime, use one of the other options, e.g., "Add External JARs..." or "Add Class Folder...". Be sure you also specify the same runtime in any associated launch configuration.

 

·        IMPORTANT: The builder as of yet does not track dependencies between compilation units, so if unit A uses something in unit B, and B changes, A does not automatically get recompiled (as it would in the JDT). If you're writing multi-file X10 applications, use "Project" -> "Clean..." as needed to get all necessary recompilations done.

 

   A subsequent release will fix this problem.

 

·        For now, the code templates in "x10.compiler/data" are not shipped as part of X10DT, since the code that would load them isn't prepared to look inside a jar file. As a result, you must have the project "x10.compiler" on your file system as a sibling of the directory "x10.common".
If you've followed the X10 "howto's" when setting up your X10 install, you already have everything you need. If not, you'll need to extract the x10.compiler project source into a directory that's a sibling of x10.common.

 

Installing X10DT:

 

To install (if you haven't installed any previous version):

 

1)      From Eclipse, invoke "Help" => "Software Updates" => "Find and Install"

2)      In the dialog that shows up, select "Search for new features to install" and click "Next"

3)      In the "Name" field type "SAFARI" (this can be anything you want, really). In the "URL" field type "http://orquesta.watson.ibm.com/eclipse/updates/SAFARI/" (note the trailing '/' character; it's significant).

4)      Click "Ok"

5)      Click "Finish"

6)      On the "Search Results" page, check the "SAFARI" checkbox, and click "Next"

7)      Click "I accept the terms..." and Click "Next"

8)      Click "Finish"; the dialog will disappear, and the status bar will show progress as the Update Manager downloads the selected updates.

9)      Click "Install All"

10)  Click "Yes" when asked whether to restart Eclipse.

 

Updating from a previous version:

 

1)      From Eclipse, invoke "Help" => "Software Updates" => "Find and Install" http://orquesta.watson.ibm.com

2)      Select "Search for updates of currently installed features"; click "Finish"

3)      When prompted for update site mirrors, just click "Cancel" to accept the main site.

4)      It often takes several minutes for Eclipse to contact the update sites and determine what updates are available.

5)      If the SAFARI site shows updates, select the new version and click "Next".

a)      Accept the license agreement and click "Finish"/"Next"

b)      Click "Install All"

c)      Click "Yes" when asked whether to restart Eclipse

 

If the SAFARI site doesn't show up in step 5 (not yet sure why it doesn't sometimes), cancel and try to install as though you hadn't previously installed (see above).

 

Installation Troubleshooting:

 

If both of the above fail, then presumably you've already installed X10DT, and you need

to manually delete "com.ibm.watson.safari" from the "features" folder of your Eclipse

installation, along with the various plug-ins: "10.compiler", "10.runtime", "x10.uide",

"polyglot2", "pe", "lpg", "org.eclipse.uide", "org.eclipse.uide.runtime", "org.jikespg.uide"

and "com.ibm.watson.safari.x10". Restart Eclipse, and install X10DT from scratch.
